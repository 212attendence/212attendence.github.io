# 2-12 출석 시스템

GitHub Pages 기반 학생 위치 출석·관리자 대시보드와 Google Apps Script 백엔드로 구성된 출석 관리 시스템입니다.

## 주요 경로

- `/` — 학생 페이지와 관리자 페이지 선택
- `/student/login/` — 학생 ID·비밀번호 로그인 및 관리자 승인 대기
- `/student/identity/` — 해강중학교 반경 100m GPS 출석 인증
- `/student/help/` — 학생 로그인·위치 권한 도움말
- `/student/error/gps/` — 위치 권한 오류 안내
- `/admin/login/` — Google 계정, 관리자 계정, 패스키를 사용하는 관리자 로그인
- `/admin/dashboard/` — 오늘 출석 현황, ESP32/MQTT 상태, 학생 로그인 승인 알림
- `/admin/help/` — 인증된 관리자 전용 도움말
- `/admin/attendance-rate/` — 오늘 출석률 상세
- `/admin/choose-login-method/` — 관리자 최초 로그인 후 패스키 설정 선택
- `/admin/setup-login-method/` — Face ID, Touch ID, Windows Hello 등 관리자 패스키 등록
- `/admin/delete-passkey/` — 관리자 패스키 삭제 확인
- `/admin/verify/` — 삭제 전 Google/관리자 재인증
- `/admin/home-icon-setting/` — PWA 설치 안내
- `/admin/no-student-added/` — 등록 학생이 없을 때의 안내

기존 경로는 새 경로로 자동 이동하도록 유지합니다.

## 학생 로그인 정책

- 학생 로그인에는 Google 계정과 패스키를 사용하지 않습니다.
- 학생 계정은 Apps Script가 생성하는 `학생계정` 시트에서 관리합니다.
- 새로운 학생 기기는 관리자 대시보드의 알림에서 승인 또는 거절합니다.
- 승인된 학생 기기 토큰은 고정 만료일이 없으며, 학생 계정 또는 `학생세션` 행을 비활성화할 때까지 유지됩니다.
- 학생이 기기 브라우저 데이터를 삭제하거나 기기를 초기화하면 다시 승인을 받아야 합니다.
- 실제 GPS 위도·경도는 출석 로그 행에 저장하지 않고 학교와의 거리, GPS 정확도, 결과만 기록합니다.

## GPS 출석 기준

- 학교명: 해강중학교
- 기준 좌표: `35°09'50.70"N 129°08'08.69"E`
- 십진수 좌표: `35.16408333333333, 129.13574722222222`
- 허용 반경: `100m`

위 기준점과 반경은 Apps Script의 고정 상수로 자동 적용됩니다. 설정 시트 값이나 관리자 기기의 현재 위치로 변경되지 않습니다.

## 관리자 세션 정책

브라우저에는 서버가 발급한 세션 토큰만 저장하며 관리자 비밀번호는 저장하지 않습니다. 세션은 최대 14일간 유지되고, 서버가 더 짧은 만료 시간을 반환하면 서버 값을 우선합니다. 로그아웃 또는 서버의 만료 응답 시 로컬 세션을 즉시 삭제합니다.

## 관리자 패스키 정책

- RP ID: `212attendence.github.io`
- 등록 및 로그인은 WebAuthn 플랫폼 인증기를 사용합니다.
- 생체 정보와 개인키는 기기 밖으로 전송되지 않습니다.
- 브라우저에는 credential ID와 등록 상태만 저장합니다.
- 서버 등록 삭제 후 운영체제나 비밀번호 관리자에 남은 패스키는 사용자가 직접 삭제해야 합니다.

## 학생 기능 백엔드

`apps-script/student-auth-addon.gs`를 기존 Apps Script 프로젝트에 추가하고 `STUDENT_LOGIN_SETUP.md` 절차에 따라 배포합니다.

생성되는 시트:

- `학생계정`
- `학생로그인요청`
- `학생세션`
- `학생출석로그`

## 화면 및 글꼴

화면 모드는 시스템, 라이트, 다크 중 선택할 수 있습니다. 글꼴 우선순위는 Google Sans → Pretendard → 운영체제 기본 한글 글꼴입니다. Google Sans가 기기에 설치되지 않은 경우 CDN의 Pretendard가 사용됩니다.

## 배포

GitHub Pages는 `main` 브랜치를 배포합니다. Apps Script 코드를 변경한 경우 GitHub 배포와 별도로 기존 웹 앱 배포를 새 버전으로 갱신해야 합니다.

## 보안 메모

- Firebase 웹 설정과 VAPID 공개키는 공개 클라이언트 식별자입니다.
- MQTT 비밀번호와 서비스 계정 개인키는 저장소에 커밋하지 않습니다.
- 관리자 계정과 학생 계정의 초기 비밀번호는 운영 전 반드시 변경해야 합니다.
- 위치 권한은 사용자가 브라우저 또는 운영체제에서 직접 허용해야 하며 웹사이트가 강제로 변경할 수 없습니다.
